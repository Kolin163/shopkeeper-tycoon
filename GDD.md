# Shopkeeper Tycoon

## 0) Питч
**Shopkeeper Tycoon** — тайм‑менеджмент/симулятор лавки. Игрок обслуживает героев, которые забегают с заказами на артефакты. Нужно быстро собрать предмет по рецепту, перетаскивая компоненты на рабочие столы. За скорость растёт комбо, за промедление клиенты уходят. Между днями игрок улучшает лавку и готовит заготовки.

**Сеттинг:** во время разработки используем Dota‑терминологию/ID/рецепты для удобства. Перед релизом — переименования/рескин.

---

## 1) Платформа и тех. ограничения
- **Движок:** Godot 4.x (Web export), язык — **GDScript**.
- **Платформа:** Yandex Games (Web).
- **Интеграция портала:** через **Playgama Bridge** (позже, отдельный этап).
- **Важное:** `res://` read‑only в экспорте; данные/сейвы — через `user://` (IndexedDB в Web).
- **Ограничения Яндекса:** архив с `index.html` в корне, имена файлов без кириллицы/пробелов, общий размер ≤ 100MB (uncompressed), корректный GameReady.

---

## 2) Основной игровой цикл (Macro Loop)
### День (DAY_ACTIVE, 3 минуты по умолчанию)
- Клиенты приходят и становятся в очередь.
- Активный клиент подходит → «озвучивает заказ» → начинается основное ожидание (serve patience).
- Игрок крафтит на столах предметы и выдаёт заказ.
- Доход + комбо + реакции клиента.

### Конец дня (DAY_END)
- Сводка: обслужено/ушло/лучшее комбо/прибыль.
- Перенос предметов со столов:
  - **базовые** → возвращаются на склад
  - **не базовые** → уходят в **заготовки (preps)**

### Перерыв (DAY_BREAK)
- Апгрейды лавки (только в перерыве).
- Мастерская/подготовка заготовок (опционально/по дизайну).
- **Ночная поставка** (подход 2): пополнение склада происходит в break.
- Кнопка `Start day`.

---

## 3) Core gameplay (Micro Loop)
1) Клиент озвучивает заказ (обычный: 1 предмет; VIP: 2–3 предмета).  
2) Игрок перетаскивает предметы со “полки” на стол.  
3) Стол автоматически собирает рецепты (комбайн).  
4) Когда нужный предмет появляется на любом столе — заказ выдается.  
5) Комбо растёт за быстрые выдачи.

---

## 4) Крафт и рецепты
### Типы предметов
- `base` — базовые компоненты, идут со склада.
- `upgrade` — крафтовые/промежуточные/итоговые предметы.
- `recipe_scroll` — отдельный предмет‑компонент (если нужен в рецепте).

### Рецепты
- **1:1 как в Dota (baseline: 7.41b)**.
- Поддержка многоуровневых рецептов (компонентом может быть `upgrade`).
- Поддержка вариативности (пример: Power Treads — выбор одного из вариантов).

### Комбайн
- При каждом drop на стол запускается автосборка: если компоненты подходят под рецепт — заменяем их на результат.
- Порядок применения рецептов должен быть **детерминированным** (например сортировка по cost+id).

---

## 5) Клиенты и очередь
### Состояния клиента
- `QUEUE` → `APPROACHING` → `ANNOUNCING` → `SERVING`
- Терпение в очереди тратится медленнее.
- Терпение в SERVING — основное (на него влияет день/апгрейды/архетип).

### Архетипы (Hero archetypes)
- `NORMAL`, `HAGGLER`, `RUSHER`, `GENEROUS`
- Влияют на:
  - терпение (mult),
  - оплату (mult),
  - чаевые (chance/mult),
  - реплику.

### Торг (Bargain v1)
- Только для `HAGGLER`, не для VIP.
- Выбор:
  - `Deal`: скидка на выплату, +терпение
  - `No`: без скидки (без бонуса)

---

## 6) VIP клиент
- Приходит **раз в 3 дня**.
- Заказ: **2–3 предмета** (список).
- Оплата: за каждый предмет + бонус за закрытие комплекта (set bonus).

---

## 7) Склад и логистика
### Stock
- Склад ведётся для базовых предметов.
- Списание происходит **на drop** на стол.
- Возврат: базовые предметы со столов возвращаются в stock при закрытии дня/сборе.

### Ночная поставка (подход 2)
- Днём склад сам по себе не пополняется.
- В перерыве применяется “night supply”:
  - базовое значение + бонус от апгрейда replenish
  - умножение от day modifier (`night_supply_mult`)

---

## 8) Дневные модификаторы (Day Modifiers)
Каждый день выбирается один модификатор, который влияет на:
- `patience_mult` — терпение в SERVING
- `pay_mult` — множитель выплаты
- `queue_drain_mult` — скорость таяния очереди
- `night_supply_mult` — ночная поставка
- веса архетипов (кого чаще спавнить)

---

## 9) Прогрессия (Progression)
- `shop_level` растёт от доставок (`deliveries_per_level`).
- `tier` у предметов ограничивает доступность.
- Пул заказов формируется так, чтобы предмет был:
  - открыт по tier
  - **реально крафтабелен** из доступных базовых (через рекурсивную проверку рецептов).

---

## 10) Апгрейды лавки (Upgrades)
- + рабочие столы (slots)
- + вместимость склада (через бонус к max stock)
- + “логистика” (replenish → усиливает ночную поставку)
- + терпение клиентов

---

## 11) Заготовки (Preps)
- Не базовые предметы со столов сохраняются в `preps` при закрытии дня.
- На старте дня `preps` раскладываются на столы (с лимитом на стол).
- Цель: игрок планирует и ускоряет будущие дни.

## 13) Сейвы
### Local Save v1
- Хранение: в editor можно писать в `res://_dev_save.json`, в Web/экспорте — `user://save.json`.
- Сохраняем:
  - gold, day_index
  - upgrades levels
  - progression state
  - stock
  - preps (список item ids)
- Точки сохранения:
  - конец дня / переход в break
  - покупка апгрейда
  - restock

### Portal Save (позже)
- Playgama/Yandex cloud save.
